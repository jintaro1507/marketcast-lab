# Marketcast Lab

過去の世界情勢・経済イベントが、各資産の価格にどう影響したかを記録・整理して可視化する、市場観察・教育目的のサイトです。

**本サイトは情報提供・教育を目的としたものであり、個別の投資助言ではありません。** 表示される数値はすべて過去の事実の記録であり、将来の値動きを示すものでも、特定の取引・銘柄・商品を推奨するものでもありません。

---

## データソース定義

各資産の価格データは、以下のソースから日次で取得しています。

| 資産 | 系列 | データソース | 備考 |
|------|------|------------|------|
| WTI原油 | DCOILWTICO | FRED | |
| 金（GLD） | gld.us | Stooq | **金現物価格ではなく、金ETF「SPDR Gold Shares（GLD）」を金価格の代理指標として使用** |
| S&P500 | SP500 | FRED | 再配布制限のため変化率/派生値のみ表示 |
| 米10年債利回り | DGS10 | FRED | 変化率(%)と絶対変化幅(pt)の両方を保持 |
| ドル円 | DEXJPUS | FRED | |
| VIX | VIXCLS | FRED | |

### 金（GLD）について

金については、従来 FRED の LBMA金価格系列（GOLDPMGBD228NLBM 等）を使用していましたが、データ提供元の都合により FRED 上で更新が停止・系列が廃止されたため、安定して取得できる金ETF「GLD」の日次価格を **金価格の代理指標** として採用しています。

GLD は金地金に裏付けられたETFで、その基準価額は LBMA金価格（PM）に連動するよう設計されているため、金価格の動き（騰落率）を見る代理指標として十分に機能します。ただし、ETFであるため、市場の需給やプレミアム/ディスカウントにより、金現物価格と完全に一致するわけではない点に留意してください。

将来的に、より適切な金現物価格の日次系列が利用可能になった場合は、そちらへの切り替えを検討します（Yahoo Finance等が代替候補）。

## 結果タグ（effect_tags）の方針

Marketcast Lab は「市場の解釈を押し付けるサイト」ではなく、「過去の事実を構造化して提示するサイト」です。

そのため、各イベントの結果タグ（effect_tags）には、**観察事実のみ**を保持します。

- 採用するタグ: `oil_up`/`oil_down`、`gold_up`/`gold_down`、`equity_up`/`equity_down`、`yield_up`/`yield_down`、`usd_strong`/`usd_weak`、`vix_up`/`vix_down`
- 採用しないタグ: `risk_on`/`risk_off` のような解釈を含むタグ

「リスクオン的だった」といった解釈は、タグではなく、編集テキスト（`why_reaction`、`key_insight`、`group_metadata`）の中で扱います。

### 結果タグの判定ルール（MVP初期閾値）

30日後の変化を基準に、以下の閾値で自動判定します。閾値未満は「横ばい」としてタグを付けません。

| 資産 | 判定方法 | 閾値 |
|------|---------|------|
| 株式・原油・金 | 相対変化率(%) | ±5%以上 |
| VIX | 相対変化率(%) | ±20%以上 |
| ドル円 | 相対変化率(%) | ±2%以上 |
| 米10年債利回り | 絶対変化幅(pt) | ±0.25pt以上 |

閾値は実データを見ながら調整する前提の初期値です。

## データ構造（三層）

- **events.json** — 個々のイベント事実。日付・カテゴリ・原因タグ・概要・個別解説（why_reaction / key_insight / contrast）・参照元を手動で管理。
- **group_metadata.json** — タグ（集合）単位の解釈。グループの概要・共通点・分岐要因・注意点・代表イベントを手動で管理。
- **event_reactions.json** — フロント表示用。ビルド時に上記2つと実データ計算結果を統合して自動生成。

---

## 自動更新

GitHub Actions（`.github/workflows/update.yml`）により、市場データの取得とイベント反応の再計算を自動実行し、GitHub Pages へデプロイします。FRED APIキーは GitHub Secrets（`FRED_API_KEY`）から注入されます。Stooq はキー不要のため追加設定は不要です。
